# 📅 [Week 2] 전문가 시연 데이터셋(Expert Dataset) 구축

**목표:** 동적 장애물을 회피하여 미션을 완수하는 고품질 시연 데이터를 수집하고, `robomimic` 호환 포맷으로 저장한다.

---

## 1. 데이터 수집 규격 (Data Specification)
팀원 간 데이터 통합 및 교차 검증을 위해 아래 규격을 엄격히 준수하십시오.

* **Task:** `NutAssembly` 또는 `PickPlace` (장애물 왕복 구간 포함)
* **포맷:** `.hdf5` (Hierarchical Data Format)
* **에피소드 수:** 최소 50개 이상의 **성공(Success)** 데이터
* **필수 Observation 키값:**
    * `robot0_joint_pos`, `robot0_joint_vel`
    * `robot0_eef_pos`, `robot0_eef_quat`
    * **`dynamic_obs_pos`**, **`dynamic_obs_vel`** (1주차 명명 규칙 준수)

---

## 2. 주요 작업 단계 (Step-by-Step)

### Step 1: Teleoperation 제어기 설정
사람이 직접 로봇을 조종하여 데이터를 쌓기 위한 스크립트를 수정합니다.


* **실행 파일:** `robosuite/scripts/collect_human_demonstrations.py` 기반 커스텀
* **Control Device:** 키보드(기본) 또는 마우스/3D 마우스 활용
* **주의사항:** 장애물의 속도가 너무 빠르면 조종이 불가능하므로, 초반에는 `speed=0.1` 정도로 낮게 설정하여 '모범 사례'를 만드는 데 집중하십시오.

### Step 2: 데이터 수집 전략 (Expert Demonstration)
단순히 성공하는 것이 아니라, RL 모델이 **'회피의 논리'**를 배울 수 있도록 수집해야 합니다.
1.  **우회 경로 다양화:** 장애물의 왼쪽으로 피하는 데이터와 오른쪽으로 피하는 데이터를 골고루 섞으십시오.
2.  **예외 상황 포함:** 장애물과 충돌 직전까지 갔다가 급하게 회피하는 '위기 탈출' 시나리오도 5~10% 포함하십시오.
3.  **정지 및 대기:** 장애물이 지나갈 때까지 잠시 멈췄다가 움직이는 데이터도 학습에 매우 유리합니다.

### Step 3: 데이터 검증 (Playback & Filter)
수집된 `.hdf5` 파일이 깨지지 않았는지, 로봇의 움직임이 부드러운지 확인합니다.
```bash
# 수집된 데이터셋 재생 및 검증
python robosuite/scripts/playback_dataset.py --dataset path/to/your/demo.hdf5
```
> **체크포인트:** 재생 시 장애물의 위치 정보가 로봇의 움직임과 동기화되어 기록되었는지 반드시 확인하십시오.
