## 2. 주요 작업 단계 (Step-by-Step)

### Step 1: 코드 클린업 및 리팩토링 (Refactoring)
다른 사람이 코드를 봤을 때 즉시 구동 가능하도록 정리합니다.
*   **Action:** 하드코딩된 경로를 상대 경로로 수정하고, `README.md`에 설치 및 실행 명령어를 명확히 기재합니다.
*   **Check:** 사양이 낮은 1/3 팀원의 PC에서도 본인의 최종 코드가 에러 없이 로드되는지 확인합니다.

### Step 3: 프로젝트 회고 (Retrospective)
기술적 성취와 한계를 비판적으로 돌아봅니다.
*   **Fact Check:** "장애물을 잘 피했다" (X) $\rightarrow$ "계층적 구조를 도입함으로써 단일 정책 대비 성공률을 $40\%$ 향상시켰으나, 엣지 디바이스에서의 연산 지연 시뮬레이션 시 성공률이 $15\%$ 하락하는 한계를 발견함" (O)
*   **Future Work:** 차후 연구(VLA 모델 통합, 실물 로봇 적용 등)로의 확장 가능성 기술.
