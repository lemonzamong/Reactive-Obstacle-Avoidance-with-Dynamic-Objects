5주차는 4주차에서 정의한 하위 스킬(Low-level Skills)들을 실제로 학습시키고, 각 스킬이 독립적으로 얼마나 잘 작동하는지 **'단위 테스트(Unit Testing)'**를 완료하는 단계입니다. 계층적 구조를 쌓기 전, 부품(Skill)들이 튼튼한지 검증하는 과정입니다.

# 📅 [Week 5] Low-level Skill Policy 학습 및 단위 검증

**목표:** 분리된 데이터셋을 사용하여 각 하위 스킬 정책을 개별적으로 최적화하고, 특정 상황에서의 성공률을 정량적으로 산출한다.

---

## 1. 스킬별 학습 전략 (Skill-Specific Training)
모든 스킬을 똑같은 방식으로 학습시키지 말고, 각 스킬의 특성에 맞게 하이퍼파라미터를 조정합니다.

*   **Approach Skill (접근):** 물체와의 거리($L2$ norm)가 줄어드는 방향으로 정확도에 집중.
*   **Avoidance Skill (회피):** 장애물의 속도 벡터($v_{obs}$)에 민감하게 반응하도록 RNN의 `horizon`이나 `history` 길이를 조절.
*   **Grasp/Release (조작):** 그리퍼의 상태 변화(Open/Close) 시점의 정밀도 확인.

---

## 2. 주요 작업 단계 (Step-by-Step)

### Step 1: Filtered Dataset 기반 개별 학습
4주차에 커팅한 데이터를 `robomimic`에 넣고 각 스킬별 모델(`.pth`)을 생성합니다.
*   **Action:** `config` 파일에서 `data.filter_key`를 사용하여 특정 스킬 데이터만 학습되도록 설정.
*   **Check:** 학습 로그에서 Loss가 수렴하는지 확인하고, 특히 **회피 스킬**의 경우 장애물과의 충돌 횟수가 줄어드는지 모니터링합니다.

### Step 2: 스킬별 독립 롤아웃 (Individual Rollout)
상위 정책 없이, 특정 조건(장애물이 앞에 있는 상황 등)을 강제로 만들고 해당 스킬이 발동되었을 때 잘 수행하는지 테스트합니다.
*   **Avoidance 테스트:** 로봇이 물체를 들고 있는 상태에서 장애물을 던져주었을 때, 충돌하지 않고 궤적을 수정하는지 확인.
*   **Grasp 테스트:** 물체 앞에서 그리퍼가 헛손질하지 않고 정확히 쥐는지 확인.

### Step 3: 성능 지표(Metrics) 산출
객관적인 데이터로 팀원들과 성과를 공유합니다.
*   **Success Rate:** 스킬 단독 수행 시 20회 중 성공 횟수.
*   **Min Distance:** 회피 스킬 작동 시 장애물과의 최소 근접 거리.
*   **Settling Time:** 스킬이 목표를 달성하는 데 걸리는 평균 시간.
