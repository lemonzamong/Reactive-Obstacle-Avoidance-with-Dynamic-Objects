# 📅 [Week 7] Hierarchical RL 통합 및 상위 정책 학습

**목표:** 고정된 하위 정책(Frozen Low-level) 위에 상위 정책(High-level Manager)을 얹어 전체 미션의 흐름을 제어하는 계층적 구조를 완성한다.

---

## 1. 계층적 구조 아키텍처 (HRL Architecture)
`robomimic`의 **HBC (Hierarchical Behavioral Cloning)** 또는 커스텀 **Switching Logic**을 구현합니다.

*   **High-level Manager (The Brain):**
    *   **입력:** 전체 환경의 저차원 상태($s_t$) 또는 카메라 이미지.
    *   **출력:** 현재 실행할 스킬의 ID(Discrete) 또는 하위 정책에 전달할 목표 지점($g_t$, Continuous).
*   **Low-level Skills (The Muscles):** 
    *   6주차까지 완성된 모델들. **학습 중단(Frozen)** 상태로 유지하여 상위 정책 학습 시 파라미터가 변하지 않도록 고정합니다.

---

## 2. 주요 작업 단계 (Step-by-Step)

### Step 1: 상위 정책 데이터 레이블링 (Meta-Data Preparation)
전문가 시연 데이터에서 "어느 시점에 스킬이 전환되었는지"에 대한 정보를 상위 정책이 배울 수 있도록 가공합니다.
*   **Action:** 4주차에 커팅했던 타임스탬프를 기반으로, 각 프레임에 `active_skill_id` 레이블을 부여합니다.
*   **Check:** 시각화 툴을 통해 로봇이 장애물 근처에 갈 때 레이블이 'Avoid'로 정상적으로 바뀌는지 팩트 체크합니다.

### Step 2: Manager Policy 학습 (Imitation 기반)
먼저 전문가의 스킬 전환 타이밍을 그대로 따라 하도록 모방 학습(BC)을 진행합니다.
*   **Action:** `robomimic`의 `HBC` 알고리즘을 사용하여 Manager를 학습시킵니다.
*   **Goal:** Manager가 장애물의 위치와 속도를 보고 적절한 타이밍에 '회피 스킬'로 스위칭하는지 확인합니다.

### Step 3: 통합 모델 롤아웃 (Full Task Rollout)
처음으로 [접근 -> 파지 -> 회피 -> 운반]으로 이어지는 전체 시나리오를 자율적으로 수행해 봅니다.
*   **분석 포인트:** 스킬이 교체되는 '경계 지점(Transition)'에서 로봇이 멈칫하거나 물체를 놓치지 않는지 집중 관찰합니다.

    *   스킬 전환 시 발생하는 지연 시간(Latency) 측정.
    *   전환 실패로 인한 에러(예: 잘못된 스킬 선택) 발생 횟수 보고.
